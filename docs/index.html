How it works

Two lines of code.
Full compliance coverage.

Agent Provenance wraps your existing agent logic and automatically captures the data lineage behind every decision.

01
Install the SDK
pip install agent-provenance. Works with LangChain, CrewAI, AutoGen, and any custom agent framework.
02
Wrap your agent
Add a ProvenanceTracer context manager around your agent call. No refactoring required.
03
Declare data assets
Tell the SDK what data your agent used — source URI, quality score, PII flag. One line per data source.
04
Generate reports
Export PDF compliance reports for legal teams, or query the API for real-time audit trail data.
python 
from agent_provenance import ProvenanceTracer, DataAssetRef

tracer = ProvenanceTracer(api_key="your-api-key")

with tracer.trace("loan-approval") as decision:
    decision.add_data_asset(DataAssetRef(
        asset_id="credit-bureau-api",
        source_uri="https://api.creditbureau.com/v2",
        quality_score=0.91,
        pii_detected=True,
        eu_ai_act_category="high_risk"
    ))
    result = your_agent.run(applicant_data)
    decision.set_output(result, model_name="gpt-4o")

# decision.record is now a compliance-ready audit trail

SDK Documentation

Get started in 2 minutes.

Everything you need to add compliance-ready audit trails to your AI agent.

pip install agent-provenance
npm install @agent-provenance/js

Requires Python 3.9+ or Node.js 18+. Compatible with all major agent frameworks.

python 
from agent_provenance import ProvenanceTracer, DataAssetRef

tracer = ProvenanceTracer(api_key="your-api-key")

with tracer.trace("loan-approval-agent") as decision:
    decision.add_data_asset(DataAssetRef(
        asset_id="credit-bureau-api",
        source_uri="https://api.creditbureau.com/v2",
        quality_score=0.91,
        record_count=1,
        pii_detected=True,
        eu_ai_act_category="high_risk"
    ))
    result = your_agent.run(applicant_data)
    decision.set_output(result, model_name="gpt-4o")

# decision.record is now a compliance-ready audit trail
python 
from agent_provenance.integrations.langchain import ProvenanceCallbackHandler
from langchain.chat_models import ChatOpenAI
from langchain.chains import LLMChain

tracer = ProvenanceTracer(api_key="your-api-key")
handler = ProvenanceCallbackHandler(tracer=tracer)

llm = ChatOpenAI(callbacks=[handler])
chain = LLMChain(llm=llm, prompt=your_prompt)

# Every chain.run() call now generates a provenance record
result = chain.run(input_data)
python 
# Track multi-agent chains with parent_decision_id
with tracer.trace("orchestrator-agent") as parent:
    parent.add_data_asset(DataAssetRef(
        asset_id="customer-db",
        quality_score=0.88
    ))

    # Child agent inherits lineage
    with tracer.trace("cv-screening-agent",
                      parent_decision_id=parent.record.decision_id) as child:
        child.add_data_asset(DataAssetRef(
            asset_id="cv-parser-api",
            source_agent_id=parent.record.decision_id,
            quality_score=0.74
        ))
        child.set_output(screening_result)

About

I built this because I kept getting the same question.

For nine years at Sage, I was VP AI Growth Innovation — which in practice meant being the person in the room when an AI agent made a bad decision and everyone looked at me to explain why.

The honest answer, most of the time, was: "I don't know. The model decided. The data was... somewhere."

I shipped Copilot features that didn't land. I rebuilt them. I sat in Cambridge IfM rooms explaining to researchers why AI agents fail at scale. I stood on stages at the British Business Awards holding a trophy for "Accountancy Software of the Year" while privately knowing that our agents couldn't explain their own decisions.

"The EU AI Act doesn't care that your model is accurate. It cares that you can prove, decision by decision, what data your agent used and whether it was trustworthy."

Agent Provenance is the tool I wish I'd had. It's open source because compliance infrastructure should be a public good — not a £50,000 consulting engagement.

I'm Anusha Su-Thirkettle. I also run HAAT Consulting, where I help companies build AI products that are worth trusting.
